FIND & SEARCH: Locating Text Position

The **FIND** and **SEARCH** functions both return the **starting position** of a text string (the `find_text`) within another text string. They are most useful when combined with text extraction functions like **MID**.

**Key Difference:** **FIND** is **case-sensitive** and does not allow wildcards. **SEARCH** is **not case-sensitive** and allows wildcards (* and ?).

---

1. FIND Function (Case-Sensitive)

=FIND(find_text, within_text, [start_num])
  • find_text: The text you want to find (e.g., `"@"` or `"Code"`).
  • within_text: The text containing the text you want to find (e.g., cell `A2`).
  • [start_num]: *Optional*. The character position to start the search from (default is 1).

Example üí° (FIND)

If A2 contains **"North-ID-123"**, this finds the position of the hyphen. Since FIND is case-sensitive, searching for "id" would fail.

=FIND("-", A2) → Result: **6**
---

2. SEARCH Function (Not Case-Sensitive)

=SEARCH(find_text, within_text, [start_num])
  • find_text: The text you want to find.
  • within_text: The text containing the text you want to find.
  • [start_num]: *Optional*. The character position to start the search from (default is 1).

Example üí° (SEARCH)

If B2 contains **"Apple - Gala"**, this finds the position of "gala" even though the case is different.

=SEARCH("gala", B2) → Result: **10**
---

Combined Dynamic Extraction Example

If A2 contains an email **"john.doe@company.com"**. You want to extract the company name ("company") using MID and FIND:

=MID(A2, FIND("@", A2)+1, FIND(".", A2)-FIND("@", A2)-1)
